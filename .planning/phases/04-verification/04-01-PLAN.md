**Test class: TestRoundTripSync**

`test_accounts_3_cycle_round_trip(in_memory_db, temp_directory)`:
- Create an Account in in_memory_db: Account(name="Cycle Test", slug="cycle-test", beginningBalance=500.0, hidden=False)
- Commit. Record all field values on the original object.
- For cycle in range(3):
  - Call export_accounts(session, temp_directory) → writes accounts.yaml
  - Read accounts.yaml with yaml.safe_load
  - Call import_accounts_from_yaml(yaml_data, session) → re-imports into same session
  - Re-query Account by slug "cycle-test"
  - Assert: name, beginningBalance, hidden match original values
  - Assert: slug is still "cycle-test" (unchanged)
- Assert: accounts.yaml exists in temp_directory after all cycles.

`test_records_3_cycle_round_trip(in_memory_db, temp_directory, sample_account, sample_category)`:
- Create a Record: label="Cycle Coffee", amount=4.50, date=datetime(2026, 3, 14), accountId=sample_account.id, categoryId=sample_category.id, isIncome=False, isTransfer=False
- Flush. The export layer generates slug via slug_generator — call export_records_for_month(session, temp_directory, 2026, 3) first to get the slug assigned (records may need to be committed before export reads them). Commit. Re-query the record. Capture record.slug — assert it matches pattern r_2026-03-14_\d{3} (use re.match).
- For cycle in range(3):
  - yaml_path = export_records_for_month(session, temp_directory, 2026, 3)
  - Read yaml_path with yaml.safe_load
  - Call import_records_from_yaml(yaml_data, session)
  - Re-query Record by slug (query by slug captured above)
  - Assert: label, amount, isIncome, isTransfer match
  - Assert: slug unchanged across cycles
- Assert: records/2026-03.yaml exists under temp_directory.

`test_slug_preservation_across_cycles(in_memory_db, temp_directory, sample_account, sample_category)`:
- Create 3 records on 2026-03-15: labels "Slug Test A", "Slug Test B", "Slug Test C". Export. Capture all 3 slugs. Run 3 round-trip cycles. After each cycle, assert all 3 slugs still exist in the re-imported DB and map to the same labels.

**Test class: TestCorruptYaml**

`test_corrupt_yaml_fails_gracefully(in_memory_db, temp_directory, sample_account)`:
- Export accounts.yaml to temp_directory.
- Read the file content. Prepend conflict markers to the content:
  ```
  <<<<<<< HEAD\nconflict content\n=======\n
  ```
  overwrite accounts.yaml with this corrupted content.
- Attempt yaml.safe_load on the corrupted content → this will either fail with yaml.YAMLError or return garbage data.
- If yaml.safe_load raises yaml.YAMLError: assert the error is a yaml.YAMLError (test passes — graceful failure).
- If yaml.safe_load returns a non-None value: call import_accounts_from_yaml(garbage_data, in_memory_db) and assert it raises either ValidationError, yaml.YAMLError, or Exception — any exception is acceptable. The database must not be silently corrupted.
- In either case: re-query the original account by slug — assert it is still present (DB integrity preserved).
- Note: The validator currently does NOT detect conflict markers directly. The corrupt YAML test verifies the system fails safely (either parse error or validation error) without silent corruption.
