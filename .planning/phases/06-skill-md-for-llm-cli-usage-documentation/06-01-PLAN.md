Create `SKILL.md` at the repository root with the following sections in this order:

**Section 1: Introduction (2-3 sentences)**
- Bagels is a local-first personal finance tracker backed by SQLite.
- The CLI provides non-interactive, structured-output commands safe for LLM use via Bash tool.
- Running `bagels` with no subcommand launches the interactive TUI (not for LLM use).

**Section 2: Global Flags**
Table with these exact entries:
| Flag | Type | Description |
|------|------|-------------|
| `--at PATH` | path | Override data/config root directory. Place before any subcommand. |

Do NOT document `--migrate` or `--source` (one-time migration tools, not for LLM workflows).

**Section 3: LLM Entry Point — `bagels llm context`**
- Purpose: Dump a complete financial snapshot as YAML to stdout.
- Mutual exclusion rule: only one of `--month`, `--period`, `--days` at a time. Defaults to current month.
- Flag table:
  | Flag | Short | Type | Default | Description |
  |------|-------|------|---------|-------------|
  | `--month` | `-m` | `YYYY-MM` | current month | Month for context dump |
  | `--period` | — | `all\|30d\|60d\|90d` | — | Named time period |
  | `--days` | — | int | — | Number of recent days |
- Output includes: snapshot_date, period, accounts, summary (income/expenses/net/record_count), spending_by_category, recent_records (capped at 30), budget_status, categories.
- Example: `bagels llm context --month 2026-03`

**Section 4: Schema Commands**

`bagels schema full` — No flags. Outputs full YAML schema for all models.
Example: `bagels schema full`

`bagels schema model MODEL_NAME` — Positional arg: `account|category|person|record|template`. Flag: `--format/-f` (`yaml|json`, default yaml).
Example: `bagels schema model record --format json`

**Section 5: Query Commands (read-only, structured output)**

For each command below, include: purpose (one line), flag table (Flag | Short | Type | Default | Description), one example.

5a. `bagels summary`
- Flags: `--month/-m` (YYYY-MM, default current), `--format/-f` (table|json|yaml, default table)
- Output fields (json/yaml): month, total_income, total_expenses, net_savings, record_count
- Example: `bagels summary --month 2026-03 --format yaml`

5b. `bagels records list`
- Flags: `--category/-c` (str), `--month/-m` (YYYY-MM), `--date-from` (YYYY-MM-DD), `--date-to` (YYYY-MM-DD), `--amount` (str, format `low..high`), `--account/-a` (str), `--person/-p` (str), `--format/-f` (table|json|yaml, default table), `--limit` (int, default 50), `--all` (flag, disables limit)
- Note: only non-transfer records returned, ordered by date descending
- Example: `bagels records list --month 2026-03 --format yaml`

5c. `bagels records show RECORD_ID`
- Positional: integer ID or slug (e.g. `r_2026-03-14_001`)
- Flag: `--format/-f` (table|json|yaml, default table)
- Example: `bagels records show r_2026-03-14_001 --format yaml`

5d. `bagels accounts list`
- Flag: `--format/-f` (table|json|yaml, default table)
- Note: hidden accounts excluded
- Example: `bagels accounts list --format yaml`

5e. `bagels categories tree`
- Flag: `--format/-f` (table|json|yaml, default table)
- Output fields: id, name, nature, color, depth, parent_id
- Example: `bagels categories tree --format yaml`

5f. `bagels spending by-category`
- Flags: `--month/-m` (YYYY-MM, default current), `--format/-f` (table|json|yaml, default table)
- Output fields: month, total, categories [{category, amount, percentage}]
- Example: `bagels spending by-category --month 2026-03 --format yaml`

5g. `bagels spending by-day`
- Flags: `--month/-m` (YYYY-MM, default current), `--format/-f` (table|json|yaml, default table)
- Output fields: month, total, daily_average, days [{date, amount}]
- Example: `bagels spending by-day --month 2026-03 --format yaml`

5h. `bagels trends`
- NOTE: `--months` (plural, int 1-12, default 3) means "how many months of history" — NOT the same as `--month YYYY-MM` on other commands.
- Flags: `--months/-m` (int 1-12, default 3), `--category/-c` (str), `--format/-f` (table|json|yaml, default table)
- Example: `bagels trends --months 6 --format yaml`

**Section 6: Mutation Commands**

`bagels records add --from-yaml PATH`
- Accepts YAML file in three formats: list of dicts, dict with `records` key, dict keyed by slugs
- Required fields: `label` (str), `amount` (float), `date` (YYYY-MM-DD)
- Optional fields: `accountSlug` (str), `categorySlug` (str), `personSlug` (str), `isIncome` (bool, default false), `isTransfer` (bool, default false)
- WARNING: May prompt interactively if some records fail validation. Pre-validate YAML before calling.
- Example showing the cat-then-add pattern:
```bash
cat > /tmp/new-record.yaml << 'EOF'
- label: "Lunch at MBK"
  amount: 245.00
  date: "2026-03-22"
  accountSlug: "kasikorn-checking"
  categorySlug: "food-dining-out"
  isIncome: false
EOF
bagels records add --from-yaml /tmp/new-record.yaml
```

**Section 7: Utility Commands**

`bagels init` — Initialize config, data directory, and SQLite database. No flags.
`bagels locate (config|database)` — Print the path to config file or database file.

**Section 8: Workflow Patterns**

Four named workflows with exact shell commands:

1. **Monthly Financial Snapshot**
   When: Get a complete picture of one month's finances.
   ```bash
   bagels llm context --month 2026-03
   ```

2. **Spending Analysis**
   When: Analyze spending patterns for a month with detailed records and category breakdown.
   ```bash
   bagels records list --month 2026-03 --format yaml
   bagels spending by-category --month 2026-03 --format yaml
   ```

3. **Budget Check**
   When: Compare actual spending against budget targets.
   ```bash
   bagels summary --month 2026-03 --format yaml
   bagels categories tree --format yaml
   ```

4. **Add a Record from LLM**
   When: Create expense records from natural language user intent.
   ```bash
   cat > /tmp/new-record.yaml << 'EOF'
   - label: "Lunch at MBK"
     amount: 245.00
     date: "2026-03-22"
     accountSlug: "kasikorn-checking"
     categorySlug: "food-dining-out"
     isIncome: false
   EOF
   bagels records add --from-yaml /tmp/new-record.yaml
   ```

**Section 9: Tips**
- Use `--format yaml` or `--format json` on any query command for machine-readable output.
- The `--at` flag goes before the subcommand: `bagels --at /path llm context`.
- `bagels llm context` is the single best command for getting full financial context in one call.

- Keep prose minimal — tables and code blocks are what LLMs parse fastest

- Do NOT use `bagels query` prefix — commands are registered directly on root group
