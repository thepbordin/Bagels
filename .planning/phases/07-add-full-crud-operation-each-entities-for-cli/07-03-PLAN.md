Change the Click option from `@click.option("--from-yaml", "-f", ...)` to `@click.option("--yaml", ...)` (no short flag for --yaml, or use `--yaml/-y` if desired). Keep the renamed `--yaml` option. Add these additional options:
- `--label` (str, default=None) — Record label/description
- `--amount` (float, default=None) — Amount (must be > 0)
- `--date` (str, default=None) — Date in YYYY-MM-DD format
- `--account-id` (int, default=None) — Account ID
- `--category-id` (int, default=None) — Category ID (optional)
- `--person-id` (int, default=None) — Person ID (optional)
- `--income` (is_flag, default=False) — Mark as income record
- `--transfer` (is_flag, default=False) — Mark as transfer
- `--transfer-to-account-id` (int, default=None) — Destination account for transfers
- `--format/-f` (choices=["table", "json", "yaml"], default="table") — Output format for created record. Now uses `-f` short flag consistently with all other commands.

**Modify the `add_record` function signature** to accept all new parameters.

**Modify the function logic:**
1. If `--yaml` is provided, run the existing YAML import logic (unchanged, just parameter name updated from `from_yaml` to `yaml`).
2. If `--yaml` is NOT provided, do inline creation:
   a. If `--label` is None, prompt: `click.prompt("Record label")`
   b. If `--amount` is None, prompt: `click.prompt("Amount", type=float)`
   c. If `--date` is None, prompt: `click.prompt("Date (YYYY-MM-DD)", default=datetime.now().strftime("%Y-%m-%d"))`
   d. If `--account-id` is None, prompt: `click.prompt("Account ID", type=int)`
   e. Validate date format: `datetime.strptime(date, "%Y-%m-%d")` — catch ValueError and raise ClickException.
   f. Validate account exists: query `session.query(Account).filter(Account.id == account_id).first()`. If None, raise ClickException.
   g. If category_id provided, validate category exists.
   h. If person_id provided, validate person exists.
   i. If transfer is True and transfer_to_account_id is None, raise ClickException("--transfer-to-account-id required when --transfer is used").
   j. If transfer_to_account_id provided, validate that account exists.
   k. Build record data dict:
      ```python
      record_data = {
          "label": label,
          "amount": float(amount),
          "date": record_date,  # the parsed datetime
          "accountId": account_id,
          "isIncome": income,
          "isTransfer": transfer,
      }
      ```
      Add `"categoryId": category_id` if not None.
      Add `"personId": person_id` if not None.
      Add `"transferToAccountId": transfer_to_account_id` if not None.
   l. Generate slug: `record_data["slug"] = generate_record_slug(record_date, session)` — import from `bagels.export.slug_generator` (already imported in the file).
   m. Import `create_record` from `bagels.managers.records`. Call `record = create_record(record_data)`.
   n. Reload the record with eager-loaded relationships for formatting: query the record back from session with joinedload on category, account, transferToAccount.
   o. Format with `format_records([record], format)` and echo.

**Important:** The renamed `yaml` parameter in the function signature corresponds to `--yaml` (Click converts `--yaml` to `yaml`). Keep all the YAML import logic intact. The new inline path is an `else` branch: `if yaml: [existing code] else: [inline creation]`.

  <action>
Add two new subcommands to the `records` group in `src/bagels/cli/records.py`:

**`records update` command:**
- `@records.command("update")`
- Positional: `identifier` (str) — integer ID or slug
- Options: `--label` (str, default=None), `--amount` (float, default=None), `--date` (str, default=None), `--account-id` (int, default=None), `--category-id` (int, default=None), `--person-id` (int, default=None), `--income/--no-income` (default=None), `--transfer/--no-transfer` (default=None), `--transfer-to-account-id` (int, default=None)
- `--format/-f` option: choices `["table", "json", "yaml"]`, default `"table"`
- Logic:
  1. `load_config()`, `init_db()`, `_open_session()`
  2. Import `resolve_entity` from `_helpers`, `Record` from models
  3. Resolve entity via `resolve_entity(session, Record, identifier)`
  4. If None, raise `ClickException(f"Record '{identifier}' not found")`
  5. Build update dict from non-None options: label -> "label", amount -> "amount", account_id -> "accountId", category_id -> "categoryId", person_id -> "personId", income -> "isIncome", transfer -> "isTransfer", transfer_to_account_id -> "transferToAccountId"
  6. If date is not None: parse with `datetime.strptime(date, "%Y-%m-%d")`, add to update dict as "date"
  7. If update dict is empty: raise `ClickException("No fields to update. Use --label, --amount, --date, --account-id, --category-id, --person-id, --income, or --transfer.")`
  8. Validate FKs if provided: account_id -> Account exists, category_id -> Category exists, person_id -> Person exists, transfer_to_account_id -> Account exists
  9. Import `update_record` from `bagels.managers.records`. Call `updated = update_record(record.id, update_data)`
  10. Reload with eager-loaded relationships for formatting. Format with `format_records([updated], format)` and echo.

**`records delete` command:**
- `@records.command("delete")`
- Positional: `identifier` (str)
- Options: `--force` (is_flag, default=False)
- Logic:
  1. `load_config()`, `init_db()`, `_open_session()`
  2. Resolve entity via `resolve_entity(session, Record, identifier)`
  3. If None, raise ClickException
  4. Import `confirm_delete` from `_helpers`
  5. Build display string: `f"{record.label} (${record.amount:.2f}, {record.date.strftime('%Y-%m-%d') if record.date else 'no date'})"`
  6. If not `confirm_delete("record", display_str, force)`: `click.echo("Cancelled.", err=True)` and return
  7. Import `delete_record` from `bagels.managers.records`. Call `delete_record(record.id)`. Note: records use HARD delete (session.delete), not soft delete — this is an intentional exception per CONTEXT.md.
  8. Echo `f"Deleted record '{record.label}' (ID: {record.id})"` to stdout.

Note: Cancellation messages must use `click.echo("Cancelled.", err=True)` — stderr per the locked decision "Error output to stderr, success output to stdout."
