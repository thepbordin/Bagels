**Create `src/bagels/cli/_helpers.py`** with these functions:

1. `resolve_entity(session, model_class, identifier: str)` — Takes a string identifier (positional arg from CLI). Try `int(identifier)` first and query by `model_class.id == int_val`. If ValueError, query by `model_class.slug == identifier`. Return the entity object or None. model_class is the SQLAlchemy model (Account, Category, Person, Record, RecordTemplate).

2. `confirm_delete(entity_name: str, entity_display: str, force: bool) -> bool` — If `force` is True, return True immediately. Otherwise call `click.confirm(f"Delete {entity_name} '{entity_display}'? ", default=False)` and return the result.

3. `check_cascade_records(session, model_class_name: str, entity_id: int) -> int` — Count records linked to this entity. For "Account": count `Record` where `accountId == entity_id`. For "Category": count `Record` where `categoryId == entity_id`. For "Person": count `Record` joins to `Split` where `Split.personId == entity_id`. Return the count integer.

4. `echo_entity(entity_obj, format_fn, output_format: str)` — Call `format_fn([entity_obj], output_format=output_format)` and `click.echo()` the result. This is the standard post-create/update echo pattern.

All functions import click at module level. The file should NOT import any model classes at module level (import inside functions to avoid circular imports).

**Extend `src/bagels/queries/formatters.py`** — Add:

1. `format_persons(persons: list, output_format: str = "table") -> str` — Same pattern as format_accounts. Table columns: ID (person.id), Name (person.name), Slug (person.slug). JSON/YAML use `_person_to_dict`.

2. `format_templates(templates: list, output_format: str = "table") -> str` — Table columns: ID (template.id), Label (template.label), Amount (template.amount formatted as `${amount:.2f}`), Account (template.account.name if template.account else "None"), Category (template.category.name if template.category else "None"), Income (Yes/No from template.isIncome), Transfer (Yes/No from template.isTransfer). JSON/YAML use `_template_to_dict`.

3. `_person_to_dict(person) -> dict` — Keys: id, name, slug.

4. `_template_to_dict(template) -> dict` — Keys: id, slug, label, amount, order, is_income (template.isIncome), is_transfer (template.isTransfer), account (dict with id+name or None), category (dict with id+name or None), transfer_to_account (dict with id+name or None from template.transferToAccount).

Import `Person` from `bagels.models.person` and `RecordTemplate` from `bagels.models.record_template` at the top of formatters.py alongside existing model imports.

- `@accounts.command("delete")`
- Positional argument: `identifier` (type=str)

If cascade and count > 0: soft-delete linked records by importing Record model and setting `deletedAt=datetime.now()` on each record where `accountId == account.id` in the session, then commit. Echo `f"Deleted account '{account.name}' (ID: {account.id})"` to stdout. If cascade was used, also echo `f"Soft-deleted {count} linked record(s)."`.

Note: For the delete command, the `--force` short flag is `-F` (capital F) since `-f` is already used by `--format` on other commands. Actually, delete does not have `--format` so `-f` can be used for `--force` here. Use `--force` with no short flag to avoid confusion, or use `@click.option("--force", is_flag=True)` without a short form.
