    - Test 4: `bagels git log` with commits prints hash + message lines
    - Test 5: `bagels git log` with no commits / no repo prints "No commits" message, exits 0

    - Test 7: `bagels git sync` when push fails prints warning but exits 0 (non-blocking)
    - Test 8: `bagels git pull` sequence: export → commit → pull → reimport (verify call order with mocks)
    - Test 9: `bagels git pull` when pull fails prints error message, exits non-zero
    - Test 10: `bagels git` (no subcommand) prints help text listing all four subcommands
  </behavior>
  <implementation>
    Step 1 — Create src/bagels/cli/git.py:

    `@click.group()` function named `git` with docstring "Git sync operations for YAML data."

    `@git.command() status`:
    - Lazy-import get_status from bagels.git.operations
    - If empty list: click.echo("Working tree clean — no uncommitted changes.")
    - Else: click.echo each file path. Print count summary.
    - No init_db() needed (read-only git operation)

    `@git.command() log`:
    - Optional `--count` option (default 10, type int)
    - Lazy-import get_log from bagels.git.operations
    - If empty list: click.echo("No commits found.")
    - Else: format each entry as "{hash}  {date}  {message}" and click.echo
    - No init_db() needed

    `@git.command() sync`:
    - Lazy-import: get_status, auto_commit_yaml, push_to_remote from bagels.git.operations
    - Lazy-import: CONFIG from bagels.config, data_directory from bagels.locations
    - Call init_db() (needed to load config via load_config() pattern from __main__.py — check existing CLI commands for correct startup sequence)
    - Get all dirty files via get_status()
    - For each file path in status, call auto_commit_yaml(data_directory() / file_path, message=f"sync: {file_path}")
    - Call push_to_remote(CONFIG.git.remote, CONFIG.git.branch) — print warning if returns False, but do not exit non-zero
    - Print summary: "Synced N files, pushed to {remote}"

    `@git.command() pull`:
    - This is the safest sequence to avoid data loss (per RESEARCH.md Pitfall 5):
      1. init_db() and load_config()
      2. Export all current SQLite entities to YAML (full export — call the same logic as `bagels export`)
      3. Commit any changes from that export
      4. Call pull_from_remote(CONFIG.git.remote, CONFIG.git.branch, silent=False) — if fails, print error and sys.exit(1)
      5. Re-import YAML into SQLite via run_full_import()
      6. Print "Pull complete — YAML reimported into database"
    - Lazy-import everything inside the function to avoid circular imports

    Step 2 — Update src/bagels/__main__.py:
    - Add at the bottom of imports section: `from bagels.cli.git import git as git_command`
    - Add: `cli.add_command(git_command, name="git")`

    Step 3 — Write tests using CliRunner and mocking git operations
  </implementation>
