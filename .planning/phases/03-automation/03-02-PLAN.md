Purpose: This is the "data always stays in sync" guarantee — any write from any caller (TUI, CLI, scripts) automatically produces a matching YAML diff. Git auto-commit fires from the same background thread if configured.

    Step 1 — Add export_records_for_month to exporter.py:
    - Function signature: `def export_records_for_month(session: Session, output_dir: Path, year: int, month: int) -> Path`
    - Compute start/end date for the month (handle December wraparound for next-month start)
    - Query Record model filtering date >= start and date < end, order by date
    - Build the same YAML dict structure as export_records_by_month (slug keys, full record data)
    - Write to output_dir / "records" / f"{year:04d}-{month:02d}.yaml" — ALWAYS write, even if records list is empty (write `{records: {}}`)
    - Return the filepath

    Step 2 — Add _trigger_export_and_commit + daemon thread call to records.py:
    - Define module-level `_GIT_EXPORT_LOCK = threading.Lock()` (coordinates with operations.py lock — but since we import operations.py lazily, just use operations._GIT_LOCK from there when calling auto_commit_yaml; the lock is inside auto_commit_yaml already, so no extra lock needed here)
    - Define `_trigger_export_and_commit(record_date: datetime, operation: str, label: str) -> None`:
      - Guard: `if CONFIG is None: return`
      - Open a new Session(), call export_records_for_month(session, data_directory(), year, month), close session
      - Guard: `if not CONFIG.git.enabled or not CONFIG.git.auto_commit: return`
      - Lazy-import auto_commit_yaml from bagels.git.operations
      - Build commit message: use CONFIG.git.commit_message_format if set, else f"records({month_key}): {operation} '{label}'"
      - Call auto_commit_yaml(filepath, message)
      - Wrap entire function body in try/except Exception: logger.exception(...)
    - In create_record: after session.expunge(record), start daemon thread with target=_trigger_export_and_commit, args=(record.date, "add", record.label)
    - In update_record: after expunge, start thread with args=(record.date, "update", record.label)
    - In delete_record: capture record.date and record.label before session.delete(record), start thread after session.commit()

    Step 3 — Add _trigger_entity_export to each non-record manager (accounts, categories, persons, record_templates):
    - Same pattern: background daemon thread calling a function that:
      1. Guards CONFIG is None check
      2. Opens new Session(), calls the appropriate export_ function (export_accounts, export_categories, export_persons, export_templates), closes session
      3. If CONFIG.git.enabled and CONFIG.git.auto_commit: lazy-import auto_commit_yaml, commit the entity yaml file
      4. Wraps in try/except, logs failures
    - Hook into create/update/delete methods of each manager
